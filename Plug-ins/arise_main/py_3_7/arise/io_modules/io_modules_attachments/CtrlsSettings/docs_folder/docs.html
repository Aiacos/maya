CtrlsSettings

The CtrlsSettings attachment exposes settings for each of the node's ctrls. You have the ability to change a ctrl shape, size, orientation, position, color, and more.

For an additional method to modify the size and shape of controls, refer to 'CtrlsShape'.

Attributes:

  • 'Ctrls Settings' - Table attribute listing every ctrl belonging to the parent Arise node.
  • 'ctrl_shape' - This dropdown offers different shapes for the ctrl. (Shapes Lookup List)
  • 'ctrl_color' - Note that changing the color will override the default logic, which sets it to yellow by default, red if the node's name starts with 'R_', and blue if it starts with 'L_'.
  • 'line_width' - Any value above 0.0 adjusts the thickness of the ctrl curve.
  • 'hide_history' - This option hides input and output nodes displayed in the ChannelBox, potentially improving Maya's performance.
  • 'ctrl_up_orient' - Rotate the ctrl shape (shape only, not the pivot). While user-friendly, it has limitations. If necessary, consider using the 'rotate_X', 'rotate_Y', and 'rotate_Z' columns.
  • 'size_X', 'size_Y', 'size_Z' - These columns scale the shape.
  • 'translate_X', 'translate_Y', 'translate_Z' - Translation offset of the shape from its pivot.
  • 'rotate_X', 'rotate_Y', 'rotate_Z' - These columns control the rotation of the ctrl shape.
  • 'tag_as_ctrl' - Specify which ctrl should be tagged as a ctrl. For example, if you set a particular ctrl shape to 'None', you may not want it to be selected alongside the other ctrls.
  • 'Reset Changes' - Reset any changes made to the 'Ctrls Settings' table above.
  • 'Update Ctrls' - If the ctrls exist in the Maya scene, this button will update them without requiring a rebuild. Any modifications made to them in the table above, including changes to the 'Ctrls Scale' attribute, will be reflected on the ctrls.
    Note that this action will override any 'CtrlsShape' attachment on the node. However, the 'CtrlsShape' will override some attributes in the 'CtrlsSettings' after a rebuild.
